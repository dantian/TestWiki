# AI導入四大維度

## 框架概述

企業AI導入需從四個整合維度同步規劃，任何單一維度的缺失都會導致導入失敗或無法持續。

```
┌─────────────────────┐    ┌─────────────────────┐
│   需求與成果         │    │   資源與投資          │
│  WHERE AI 創造價值   │    │  HOW 建立AI能力      │
└─────────────────────┘    └─────────────────────┘
┌─────────────────────┐    ┌─────────────────────┐
│      監控            │    │       治理            │
│  WHAT 衡量成效       │    │  WHO 負責任地導入    │
└─────────────────────┘    └─────────────────────┘
```

---

## 維度一：需求與成果（Requirements & Results）

**核心問題**：AI在哪裡為業務創造實質影響？

### 四大業務影響類型

| 類型 | 說明 | 典型應用 |
|------|------|---------|
| **營運效率** | 流程自動化、降低成本 | 文件自動審閱、資料整理 |
| **營收成長** | 新產品、定價優化 | 智慧定價、客戶分析 |
| **風險降低** | 供應鏈韌性、合規 | 異常偵測、合規檢核 |
| **策略差異化** | 新商業模式、AI原生流程 | 全新服務、競爭護城河 |

### 應用案例評估標準

每個AI應用案例需在五個維度上評分：
1. **策略對齊** — 是否符合公司/部門優先目標
2. **財務影響** — 可量化的成本節省或營收提升
3. **可行性與資料就緒度** — 資料是否存在、品質是否足夠
4. **風險暴露** — 合規、倫理、資安風險
5. **價值實現時間** — 快速見效 vs 長期佈局

## 維度二：資源與投資（Resources & Investment）

**核心問題**：如何建立制度性AI能力，而非零散實驗？

### 三階段能力建設路徑

| 時程 | 階段名稱 | 核心任務 |
|------|---------|---------|
| 短期 | **基礎建設** | 基礎工具採購、試點應用、核心資料整備、基本治理 |
| 中期 | **規模擴展** | 擴大成功案例、擴充基礎設施、員工培訓計畫、廠商關係深化 |
| 長期 | **深度嵌入** | AI嵌入核心營運、AI原生流程、持續學習文化、制度能力形成 |

**關鍵原則**：建立的是**制度能力**，不是一個個孤立的實驗。

## 對主管的意義

- **不能只做工具評估**：工具只是「資源與投資」維度的一部分，缺少需求定義、監控指標和治理機制，工具無法產生持續價值。
- **四個維度是主管的責任分工框架**：哪個部門負責需求？IT負責哪部分投資？誰設定監控指標？誰核准上線？
- **先從需求與成果開始**：確認業務問題，再決定技術方案，反過來做的導入成功率很低。
