# AI導入監控指標

## 為何需要監控框架？

AI投資必須持續優化。沒有指標，就無法判斷哪些應用值得擴大、哪些應該停止。監控是AI導入四大維度之一，與需求定義同等重要。

---

## 四類指標

### 財務指標（Financial）

| 指標 | 說明 |
|------|------|
| **成本節省** | 導入AI後節省的人力、時間、資源成本 |
| **營收提升** | AI直接或間接帶來的營收增加 |
| **ROI與回本期** | 投資回報率，以及何時達到損益兩平 |
| **利潤改善** | 毛利率或淨利率的改變 |

### 營運指標（Operational）

| 指標 | 說明 |
|------|------|
| **生產力提升** | 每人每小時產出量的增加 |
| **週期時間縮短** | 完成一個任務或流程所需時間的減少 |
| **錯誤率降低** | 人工錯誤、資料錯誤的減少 |
| **吞吐量增加** | 相同時間內處理更多工作量 |

### 採用指標（Adoption）

| 指標 | 說明 |
|------|------|
| **使用率** | 員工實際使用AI工具的比例與頻率 |
| **員工參與度** | 員工對AI工具的滿意度與主動使用意願 |
| **AI素養成長** | 員工理解和使用AI的能力提升 |
| **實驗率** | 員工主動嘗試新AI應用的頻率 |

### 風險指標（Risk）

| 指標 | 說明 |
|------|------|
| **合規事件** | 因AI使用引發的法規或資安事件數量 |
| **模型效能漂移** | 模型準確率隨時間下降的程度 |
| **倫理AI違規** | 偏見、不公平、不透明等倫理問題的發生次數 |
| **事件回應時間** | 從發現問題到完成處理的時間 |

---

## 領先指標 vs 落後指標

| 類型 | 定義 | 用途 | 範例 |
|------|------|------|------|
| **領先指標** | 預測未來績效 | 早期預警，及時調整 | 每週查詢成功率、使用者數量 |
| **落後指標** | 確認已實現成效 | 驗證結果，報告成果 | 季度費用節省、ROI達成率 |

**建議**：兩種指標都需要，領先指標讓你能在問題惡化前介入，落後指標讓你能向上報告成果。

---

## 具體案例中的指標設定

### 研發知識助理（參考）
- 領先：每週查詢成功率、R&D資料量、使用者數量
- 落後：季度費用節省
- 模型漂移觸發點：匹配準確率低於90%
- 目標ROI回本期：**6個月**

### 實驗顧問AI（參考）
- 領先：建議命中率、實驗工作量減少比例
- 落後：季度費用節省
- 模型漂移觸發點：模型準確率低於40%
- 目標ROI回本期：**12個月**

## 指標設定原則

1. **每個AI應用都需要自己的KPI**，不能用通用指標籠統衡量
2. **在上線前就設定指標**，上線後才設定容易被成果影響而失去客觀性
3. **設定現行基準（baseline）**，沒有基準就無法衡量改善幅度
4. **季度審查**：部門主管應每季檢視AI應用的指標達成情況

---

## 對主管的意義

- 你的AI預算申請需要附上預期KPI與回本期
- 沒有監控框架的AI導入，無法對董事會或股東說明投資邏輯
- 監控不是IT的工作——部門主管需要定義業務指標，IT只負責技術指標
