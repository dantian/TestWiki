# 研發知識助理（R&D Knowledge Assistant）

## 應用概述

**目標**：讓研究人員使用類GPT的對話介面查詢和定位所有R&D資料

**來源**：fp_aiplan_v2.pdf — 作為AI導入四大維度框架的具體示範案例

---

## 四大維度分析

### 需求與成果

| 項目 | 內容 |
|------|------|
| **核心需求** | 研究人員能以自然語言查詢、定位所有R&D資料 |
| **業務問題** | 研發資料散落於PLM、Lotus Notes、紙本等多個系統，搜尋費時 |
| **目標指標** | 搜尋時間降低**60%** |
| **KPI定義** | 在數分鐘內找到並定位R&D資料 |
| **價值實現來源** | PLM系統、Lotus Notes、紙本資料 |

### 資源與投資

| 項目 | 內容 |
|------|------|
| **技術方案** | OCR + LLM pipeline，處理現有R&D資料 |
| **第一階段工具** | Gemini NotebookLM |
| **人員培訓** | 2天培訓計畫 |
| **預估投資** | DGX Spark（地端）+ Gemini授權 |

**備註**：選擇DGX Spark地端部署，顯示此部門R&D資料屬於高機密，不適合上傳公有雲。

### 監控指標

| 類型 | 指標 |
|------|------|
| **領先指標** | 每週查詢成功率 |
| **領先指標** | R&D資料量（資料庫成長狀況） |
| **領先指標** | 使用者數量 |
| **落後指標** | 季度費用節省 |
| **模型漂移觸發點** | 匹配準確率低於**90%** |
| **目標ROI回本期** | **6個月** |

### 治理

| 項目 | 內容 |
|------|------|
| **資料管理** | AI種子負責管理R&D資料的成長與更新 |
| **品質回饋** | 使用者對查詢結果品質提供反饋 |
| **稽核機制** | 收集用戶查詢的稽核日誌 |
| **定期審查** | 部門主管每季審查績效 |

---

## 可複製到其他部門的元素

此案例展示了一個完整的AI應用設計模板：

1. **具體的業務痛點**：R&D資料散落多系統 → 可替換為其他部門的痛點
2. **量化的目標**：60%搜尋時間縮短 → 每個部門需設定自己的目標數字
3. **分階段的工具選擇**：從NotebookLM開始，有地端需求才加DGX Spark
4. **雙層指標**：領先指標（查詢成功率）+ 落後指標（費用節省）
5. **治理機制**：AI種子角色 + 稽核日誌 + 季度審查

---

## 對主管的意義

- **6個月回本期**是相對積極但合理的目標，前提是能快速整合資料來源
- **60%搜尋時間降低**等於讓研究人員有更多時間做真正的研究工作
- **此案例可作為其他部門提案的模板**，只需替換業務痛點與資料來源
